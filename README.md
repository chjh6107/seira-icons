## Usage

```tsx
import { Heart, HeartOutline, HeartSharp } from './icons';
import { LogoReact } from './icons';
import { Spinner } from './icons';

// Basic usage
<Heart />

// Resize (default is 24×24)
<HeartOutline size={24} />

// Change color (currentColor based)
<HeartSharp style={{ color: 'red' }} />
// or
<HeartSharp className="text-red-500" />
```

## Icon Variants

Each base icon comes in 3 styles.

| Variant | Suffix | Example | Description |
|---------|--------|---------|-------------|
| Filled | (none) | `<Heart />` | Solid filled style |
| Outline | `Outline` | `<HeartOutline />` | Stroke outline style |
| Sharp | `Sharp` | `<HeartSharp />` | Sharp corner style |

Logo icons (`Logo-` prefix) and Spinner are provided as a single variant only.

## CLI (copy-in)

Copy only the icons you need into your project — no runtime dependency:

    npx @seira-icons/ionicons add heart heart-outline

- Names use the icon's **kebab filename**. The **filled** variant is the bare
  name: `heart` (filled), `heart-outline`, `heart-sharp`.
- Files land in `src/components/icons` (or `components/icons` if you have no
  `src/`). Override with `--dir <path>`; the choice is saved to
  `seira-icons.json`.
- Existing files are **skipped**; pass `--overwrite` to update them (does not affect a hand-written barrel).
- A shared `icon.types.ts` and `IONICONS-LICENSE` are copied alongside.
- Brand `logo-*` icons also copy a `TRADEMARKS.md`; MIT covers the artwork,
  not the mark.

### Barrel (`index.ts`)

`add` also generates and maintains a barrel so you can import from one place:

    import { Heart, Star } from '@/components/icons';

- The `index.ts` is **managed**: it carries a `// Generated by @seira-icons/ionicons — do not edit.`
  header and is rebuilt from the folder's icons on every `add`.
- It is **additive** — deep imports (`import Heart from '@/components/icons/heart'`) keep working.
- A **hand-written** `index.ts` (without the marker) is never overwritten — even with `--overwrite`.
  To adopt it, delete it and re-run `add`.
- Opt out with `--no-barrel` (persisted to `seira-icons.json`).
- If you remove an icon file by hand, re-run any `add` to prune it from the barrel.
- Tip: add `components/icons/index.ts` to `.prettierignore` to avoid formatter churn.

**Requirements:** TypeScript, `jsx: "react-jsx"`, and `@types/react`.
`moduleResolution` `bundler` or classic `node` (extensionless relative imports;
`node16`/`nodenext` require adding `.js` extensions yourself).

## Spinner

A loading indicator designed to be used with CSS animation.

```tsx
import { Spinner } from './icons';

// With CSS animation
<Spinner
  size={32}
  style={{ animation: 'spin 1s linear infinite' }}
/>
```

```css
@keyframes spin {
  from { transform: rotate(0deg); }
  to { transform: rotate(360deg); }
}
```

## Component API

All icons accept `IconProps` — the standard `SVGProps<SVGSVGElement>` plus a `size` shorthand. Any standard SVG attribute can be used.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `size` | `number \| string` | `24` | Sets both width and height (number → px, string → any CSS length) |
| `width` | `number \| string` | `size` | Width (overrides `size`) |
| `height` | `number \| string` | `size` | Height (overrides `size`) |
| `fill` | `string` | `currentColor` | Fill color |
| `className` | `string` | - | CSS class |
| `style` | `CSSProperties` | - | Inline styles |
| `...rest` | `SVGProps` | - | Other SVG attributes |
