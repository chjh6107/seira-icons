# 배럴(`index.ts`) 자동 생성 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** `add`가 대상 폴더에 managed `index.ts`(배럴)를 결정적으로 재생성해 `import { Heart, Star } from '@/components/icons'`를 가능하게 한다(파일별 deep import 유지, additive).

**Architecture:** 새 순수/IO 분리 모듈 `src/cli/barrel.js`(export명 파싱+가드, 문자열 조립, 마커 판정, 폴더 스캔 쓰기)를 `add.js`가 아이콘 복사 후 호출한다. 사용자 `index.ts`는 마커 인지로 보호하며, 쓰기는 기존 `fs-safe.ensureFile`에 새 `allowOverwrite(existing)` predicate를 넣어 수행한다.

**Tech Stack:** Node ESM(`type:module`), `node:fs`/`node:path`, vitest, TypeScript(`checkJs`). 런타임 의존성 0.

**Spec:** `docs/superpowers/specs/2026-07-07-barrel-generation-design.md`

## Global Constraints

- 런타임 의존성 0(`node:*`만). ESM, intra-CLI import는 명시적 `.js`. `// @ts-check` + JSDoc. `tsconfig.cli.json`로 타입체크.
- 마커 상수(정확): `MARKER = "// Generated by @seira-icons/ionicons — do not edit."` (em-dash U+2014). 매칭용 `MARKER_TOKEN = "// Generated by @seira-icons/ionicons"`(ASCII).
- canonical 출력: LF 개행, 파일 끝 단일 `\n`. 정렬은 **코드유닛**(`(a,b)=>a<b?-1:a>b?1:0`), `localeCompare` 금지.
- export명 파싱 가드: 파싱된 맵 크기가 `< 1000`이면 throw(조용한 빈-배럴 방지).
- map-key 게이트: 대상 폴더 `*.tsx` 중 **맵 key인 것만** re-export, **맵 value**를 방출(인젝션/foreign 차단).
- 마커 판정: 첫 줄에서 선행 BOM(`﻿`) 제거 + 말미 `\r`/공백 제거 후 `startsWith(MARKER_TOKEN)`.
- 배럴 쓰기: `ensureFile(index.ts, content, { compare: normalizeEol, allowOverwrite: isManaged })`. **`--overwrite`는 배럴에 전달하지 않는다**(unmanaged index.ts 비클로버). `ensureFile`의 `lstat` 심링크 거부 + `O_EXCL|O_NOFOLLOW`/`O_TRUNC|O_NOFOLLOW` 불변.
- 기본 배럴 ON. `--no-barrel` 옵트아웃 → `seira-icons.json`에 `barrel:false` 지속. 설정 `barrel===false`면 배럴 off.
- 배럴 경고/상태는 stdout·exit 0. 심볼 `+`(생성)/`=`(동일)/`!`(비클로버).
- `ensureBarrel(realDir)`의 `realDir`는 호출자(`add.js`)가 `assertInsideProject`로 confine한 경로.

// @ts-check
import { readdirSync, existsSync } from 'node:fs';
import { join } from 'node:path';
import { packageFile } from './paths.js';
import { ensureFile } from './fs-safe.js';

export const MARKER = '// Generated by @seira-icons/ionicons — do not edit.';
export const MARKER_TOKEN = '// Generated by @seira-icons/ionicons';

/**
 * @param {string} src
 * @returns {Map<string,string>} kebab → Pascal
 */
export function parseExportMap(src) {
  const map = new Map();
  const re = /^export \{ default as (\w+) \} from '\.\/([\w-]+)';$/;
  for (const line of src.split('\n')) {
    const m = re.exec(line);
    if (m) map.set(m[2], m[1]);
  }
  if (map.size < 1000) {
    throw new Error(`export map parse collapsed (${map.size} entries) — package icons/index.ts format changed`);
  }
  return map;
}

/** @type {Map<string,string>|null} */
let _map = null;
/** @returns {Map<string,string>} */
export function exportNameMap() {
  if (!_map) _map = parseExportMap(packageFile('icons/index.ts'));
  return _map;
}

/**
 * @param {string[]} iconBasenames
 * @param {{hasTypes: boolean}} opts
 * @returns {string}
 */
export function buildBarrel(iconBasenames, { hasTypes }) {
  const map = exportNameMap();
  const lines = [MARKER];
  if (hasTypes) lines.push(`export type { IconProps } from './icon.types';`);
  const known = iconBasenames
    .filter((n) => map.has(n))
    .sort((a, b) => (a < b ? -1 : a > b ? 1 : 0));
  for (const n of known) {
    lines.push(`export { default as ${map.get(n)} } from './${n}';`);
  }
  return `${lines.join('\n')}\n`;
}

/** @param {string} existing @returns {boolean} */
export function isManaged(existing) {
  const firstLine = existing.replace(/^﻿/, '').split('\n', 1)[0].replace(/\r$/, '').trimEnd();
  return firstLine.startsWith(MARKER_TOKEN);
}

/** @param {string} s @returns {string} */
function normalizeEol(s) {
  return s.replace(/\r\n/g, '\n');
}

/**
 * @param {string} realDir  already confined by the caller
 * @returns {{status: 'written'|'skipped-identical'|'skipped-unmanaged', count: number}}
 */
export function ensureBarrel(realDir) {
  const map = exportNameMap();
  let basenames;
  try {
    basenames = readdirSync(realDir)
      .filter((f) => f.endsWith('.tsx'))
      .map((f) => f.slice(0, -'.tsx'.length));
  } catch {
    return { status: 'skipped-unmanaged', count: 0 };
  }
  const known = basenames.filter((n) => map.has(n));
  const hasTypes = existsSync(join(realDir, 'icon.types.ts'));
  const content = buildBarrel(known, { hasTypes });
  let status;
  try {
    status = ensureFile(join(realDir, 'index.ts'), content, {
      compare: normalizeEol,
      allowOverwrite: isManaged,
    });
  } catch {
    // symlinked / EISDIR / unreadable index.ts — never followed/clobbered; downgrade, don't fail the add
    return { status: 'skipped-unmanaged', count: known.length };
  }
  return {
    status: status === 'conflict' ? 'skipped-unmanaged' : status,
    count: known.length,
  };
}
```

### Barrel (`index.ts`)

`add` also generates and maintains a barrel so you can import from one place:

    import { Heart, Star } from '@/components/icons';

- The `index.ts` is **managed**: it carries a `// Generated by @seira-icons/ionicons — do not edit.`
  header and is rebuilt from the folder's icons on every `add`.
- It is **additive** — deep imports (`import Heart from '@/components/icons/heart'`) keep working.
- A **hand-written** `index.ts` (without the marker) is never overwritten — even with `--overwrite`.
  To adopt it, delete it and re-run `add`.
- Opt out with `--no-barrel` (persisted to `seira-icons.json`).
- If you remove an icon file by hand, re-run any `add` to prune it from the barrel.
- Tip: add `components/icons/index.ts` to `.prettierignore` to avoid formatter churn.
```
